Because we use our organisation's Azure AD, users sign in with their existing work accounts — no separate credentials. Azure already knows who they are. The backend uses the client secret to call Microsoft Graph and pull user/group data from our directory.

Wrap your app root with MsalProvider — that's it for plumbing.

import { PublicClientApplication } from '@azure/msal-browser';
import { MsalProvider } from '@azure/msal-react';

const msalInstance = new PublicClientApplication({
  auth: {
    clientId: 'YOUR_CLIENT_ID',
    authority: 'https://login.microsoftonline.com/YOUR_TENANT_ID',
    redirectUri: window.location.origin,
  }
});

export default function App() {
  return (
    <MsalProvider instance={msalInstance}>
      <YourApp />
    </MsalProvider>
  );
}

Then anywhere inside: useMsal() / useIsAuthenticated() hooks just work.

Spring Boot properties — the client secret never touches the frontend.

graph.client-id=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
graph.client-secret=your-secret-here
graph.tenant-id=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
graph.scope=https://graph.microsoft.com/.default

The .default scope means "everything the app has been granted in Azure AD" — no per-call scope juggling.

Store these in Azure Key Vault or env vars in prod — never commit to Git.

Challenge
Why did it work in dev but not in prod?
"Initially we didn't understand why it was working in dev but not working in prod."
Azure AD only accepts http://localhost over plain HTTP — everything else must be HTTPS. In dev, localhost gets a special pass. In prod, we were using an IP address or a plain HTTP URL, which Azure silently rejects.
http://192.168.1.10:3000 http://ourapp.com https://ourapp.yourdomain.com
How we fixed it
We resolved this by deploying the application under a domain with a valid SSL certificate, registering the corresponding https:// URL as the redirect URI in Azure AD.
Why SPA (not Web)?
SPA platform uses PKCE — no client secret in the browser. More secure for public clients like React apps.
Why .default scope?
Backend uses client credentials flow. .default requests everything pre-approved in Azure — cleaner than listing scopes.
Why only https in prod?
Azure AD enforces HTTPS on non-localhost URIs. It's also just good practice — tokens travel over this connection.
Why our org's tenant?
Users don't need new accounts. We get group membership and org data from our existing directory for free.
